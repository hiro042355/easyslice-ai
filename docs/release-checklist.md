# NEXCUT AI Release Checklist

## 方針

現在の構成では、公開版はアップロード動画を主軸にする。

- `/` はアプリ本体
- `/landing` は紹介ページ
- YouTube URL取得はローカル向け・実験機能として扱う
- 公開版では動画アップロード、字幕ファイル、音声ハイライト、字幕AIハイライトを中心にする

## 公開前の必須確認

### 1. 環境変数

- [ ] `.env.local` をGitHubにコミットしていない
- [ ] `GEMINI_API_KEY` を本番環境の環境変数に設定する
- [ ] Gemini APIキーをフロントエンドに書いていない
- [ ] `NEXT_PUBLIC_GEMINI_API_KEY` を使っていない

### 2. Cookie / YouTube

- [ ] YouTube Cookieを使用していない
- [ ] `YOUTUBE_COOKIES` を本番環境に設定していない
- [ ] `cookies.txt` がリポジトリに含まれていない
- [ ] YouTube URL取得は不安定な可能性があることをUI/READMEで明記している

### 3. Git / ファイル管理

- [ ] `.gitignore` に `.env*` が含まれている
- [ ] 生成された `.mp4` をGitHubに含めていない
- [ ] 生成された `.zip` をGitHubに含めていない
- [ ] `node_modules` を含めていない
- [ ] `.next` を含めていない

### 4. ビルド

- [ ] `npm run build` が成功する
- [ ] TypeScriptエラーがない
- [ ] 不要なAPIや未使用ボタンが残っていない

### 5. アップロード動画ワークフロー

- [ ] 動画ファイルをアップロードできる
- [ ] アップロード動画をプレビューできる
- [ ] 動画時間が正しく表示される
- [ ] Clipプレビューで開始秒に移動する
- [ ] 終了秒で自動停止する
- [ ] ZIP一括生成できる
- [ ] 生成されたMP4を再生できる

### 6. 字幕ファイルワークフロー

- [ ] `.txt` 字幕を読み込める
- [ ] `.srt` 字幕を読み込める
- [ ] `.vtt` 字幕を読み込める
- [ ] 動画時間外の字幕を除外できる
- [ ] 字幕AIハイライトで候補が出る
- [ ] Gemini上限時のエラー表示が分かりやすい

### 7. 音声ハイライト

- [ ] 字幕なし動画で音声ハイライト候補を生成できる
- [ ] 音声ハイライト候補をプレビューできる
- [ ] 音声ハイライト候補からZIP生成できる

### 8. YouTubeローカル機能

- [ ] YouTube URLから動画取得できる
- [ ] YouTube字幕取得ができる
- [ ] YouTube取得が失敗した場合の表示が分かりやすい
- [ ] 公開環境では実験機能として扱う

### 9. UI / UX

- [ ] `/landing` が表示できる
- [ ] `/` から `/landing` へ戻れる
- [ ] ボタン名が分かりやすい
- [ ] アップロード導線が分かりやすい
- [ ] YouTube機能がローカル向けであることが分かる
- [ ] リセットボタンで候補・結果を消せる
- [ ] エラー時に次の行動が分かる

## 公開時の推奨構成

### 推奨

- 動画アップロード
- 字幕ファイルアップロード
- 音声ハイライト
- 字幕AIハイライト
- ZIP一括生成

### 注意

YouTube URL取得は、以下の理由で公開版の主軸にしない。

- 403 / 429 が出る可能性
- yt-dlpがホスティング環境で不安定になる可能性
- Cookie利用はセキュリティリスクがある
- YouTube側の仕様変更に影響されやすい

```bash
git status
```
